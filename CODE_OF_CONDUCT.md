# Code of Conduct

## Our Commitment

ProofSettle is committed to fostering an open, professional, and respectful community where contributors, collaborators, and users can work together to advance transparent decision intelligence.

We welcome contributions from developers, researchers, designers, product thinkers, domain experts, and the broader open-source community.

---

## Expected Behavior

All participants are expected to:

- Be respectful and professional.
- Encourage constructive discussions and technical collaboration.
- Provide thoughtful feedback and code reviews.
- Respect differing viewpoints and experiences.
- Focus discussions on improving the project.
- Help create an inclusive and welcoming environment.

---

## Unacceptable Behavior

The following behaviors are not acceptable:

- Harassment, discrimination, or abusive language.
- Personal attacks or inflammatory discussions.
- Publishing confidential or proprietary information without authorization.
- Deliberate disruption of discussions or project activities.
- Any conduct that would make others feel unsafe or unwelcome.

---

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct.

They may remove, edit, or reject contributions, comments, issues, or pull requests that violate these guidelines.

---

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub Issues
- Pull Requests
- Discussions
- Documentation
- Community communications

---

## Our Philosophy

ProofSettle is built on the principle that **every important decision deserves a transparent procedure**.

We believe the same principle should guide collaboration within our community—through openness, professionalism, respect, and accountability.

---

Thank you for helping make ProofSettle a welcoming and collaborative project.
